Detailed Description

template<typename element_type>
class utilities::MathSet< element_type >

Implements an std::unordered_set-like object except that it knows about union, etc.

Note
We actually use std::set to avoid the need for requiring the elements to be hashable.
I'm not convinced that the various operations are optimal. It may be easier to do them directly with the members of std::set. Also switching to a sorted std::vector could possibly speed things up by taking advantage of the fact that it wouldn't be random memory access.
Template Parameters
element_typeThe type of the elements in the set. Must be copyable.
